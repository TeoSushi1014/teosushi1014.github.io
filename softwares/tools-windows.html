Overview of Scripts: Windows Optimization and Management Tools

These scripts are designed to perform various tasks related to Windows system optimization, maintenance, and customization. Each script targets a specific area and requires administrator privileges to function correctly.

DisableDefender.cmd

  • Purpose: Disables Windows Defender security features.
  • Actions: Modifies registry values to turn off real-time protection, cloud-delivered protection, automatic sample submission, and other components.
  • Cautions: Significantly reduces system security, making it more vulnerable to malware. Use with extreme caution and only if you have alternative security measures in place.

View details

DisableDefender.cmd

This script is designed to optimize Windows by disabling various Windows Security features. It specifically targets Virus & Threat Protection components like Real-time protection, Cloud-delivered protection, Automatic sample submission, etc.

How It Works

  1. Checks for Admin Rights: Ensures the script has the necessary permissions to modify system settings.
  2. Disables Windows Security Features:
    • Uses REG ADD commands to modify registry values, effectively turning off the listed security features.
    • Changes are applied to both 32-bit and 64-bit registry entries.
  3. Provides User Feedback:
    • Displays messages indicating the progress of the disabling process.
    • Pauses at the end to allow the user to review the output.

Important Considerations

  • Security Implications: Disabling these features significantly reduces system security and makes it more vulnerable to malware and other threats. Use with caution.
  • Administrator Privileges: Requires administrator rights to run successfully.
  • Registry Modification: Directly modifies the Windows registry, so errors could cause system instability.

EnableDefender.cmd

  • Purpose: Re-enables Windows Defender after it has been disabled.
  • Actions: Deletes specific registry keys that disable Windows Defender and restarts related services.
  • Recommendations: Keep Windows Defender enabled for basic protection unless you have a robust third-party security solution.

EnableDefender.cmd

This script is designed to re-enable Windows Security (Windows Defender) on a system where it might have been disabled, potentially by an administrator or another script.

How It Works

  1. Clears Console: The cls command clears the console screen for clean output.
  2. Re-enables Windows Defender Components:
    • Uses reg delete commands to remove specific registry keys that disable various aspects of Windows Defender:
      • Real-Time Protection
      • Antivirus engine
      • Security Center (including notifications)
  3. Starts Windows Security Services:
    • Uses net start commands to start three essential services:
      • WinDefend: The main Windows Defender service.
      • WdNisSvc: Network Inspection Service, monitors network traffic for threats.
  4. Provides User Feedback:
    • Prints a success message confirming Windows Security is enabled.
    • Notes that the system's virus protection may have been previously managed by an organization.
    • Briefly pauses for user acknowledgement.

Important Considerations

  • Administrator Privileges: Required to run this script due to system modifications.
  • Restores Default Protection: This script restores the default Windows Security settings. If you had other security software that disabled Windows Defender, you'll need to reconfigure it separately.
  • Security Best Practices: While Windows Security offers basic protection, consider additional security software for enhanced defense against threats.

EnableWindowsUpdate.cmd

This script is designed to re-enable Windows Update on your system if it has been paused or disabled.

How It Works

  1. Checks Administrator Privileges:
    • Ensures the script runs with administrator permissions required for system modifications.
    • Prompts for elevation and exits if not run as administrator.
    • Double-checks administrator privileges using net session.
  2. Deletes Registry Keys:
    • Removes specific registry keys that control Windows Update settings using REG DELETE.
    • Suppresses command output with >nul for a clean console.
  3. Provides Feedback:
    • Prints a success message: "Windows Update đã được bật lại!" (Windows Update has been re-enabled!)
    • Pauses briefly to allow the user to read the message before the console window closes.

Important Considerations

  • Administrator Rights: Absolutely essential for this script to function.
  • Registry Modification: Directly modifies the Windows registry, so exercise caution.
  • Resets Windows Update: Reverts to default settings, removing any pauses or customizations.
  • Potential Impact: Any specific update preferences you had will be lost and need to be re-configured.

Note: Before running this script, create a system restore point to revert any unwanted changes. If you intentionally disabled Windows Update, ensure you want to re-enable it.

Fast Startup.cmd

This script is designed to optimize Windows performance by adjusting certain system settings and features.

Key Actions

  1. Checks for Administrator Privileges:
    • Ensures the script has the necessary permissions to modify system settings.
    • Prompts the user to elevate privileges if not running as administrator.
  2. Displays an Intro Animation:
    • Creates a stylized text animation that says "Powered by TEO SUSHI" using ANSI escape codes for color and formatting.
  3. Enables Hibernation and Fast Startup:
    • Uses powercfg.exe to enable hibernation and fast startup features for improved boot times.
  4. Enables Sysmain Service and Superfetch:
    • Modifies the registry to set the Sysmain service to start automatically.
    • Starts the Sysmain service, which is responsible for Superfetch (preloading frequently used applications).

Important Considerations

  • Administrator Rights: Required to modify system settings.
  • Performance Optimization: Aims to improve boot times and application loading speed through hibernation, fast startup, Sysmain, and Superfetch.
  • Compatibility: Not suitable for all systems. Understand the implications before running the script.

Caveats

  • Superfetch: May cause issues on systems with limited resources. Disable if you encounter problems.
  • Hibernation: Requires significant disk space (roughly equal to your RAM). Ensure sufficient free space before enabling.

GamingMode.cmd

This script appears to be designed to optimize Windows for gaming performance by enabling Game Mode and making the Ultimate Performance power plan available.

How It Works

  1. Checks for Administrator Privileges:
    • Ensures the script has the necessary permissions to modify system settings.
    • Prompts the user to elevate privileges if not running as administrator.
  2. Displays an Intro Animation:
    • Creates a stylized text animation that says "Powered by TEO SUSHI" using ANSI escape codes.
  3. Enables Ultimate Performance Power Plan:
    • Uses powercfg -duplicatescheme e9a42b02-d5df-448d-aa00-03f14749eb61 to duplicate the GUID associated with the Ultimate Performance power plan. This makes it available on systems where it's hidden.
  4. Opens Power Options:
    • Executes powercfg.cpl to open the Power Options control panel, allowing manual selection of power plans.

Key Points

  • Administrator Rights: Required for modifying power settings.
  • Game Mode and Ultimate Performance:
    • Game Mode: Prioritizes gaming performance by optimizing system resources.
    • Ultimate Performance: Maximizes performance by disabling power-saving features (may increase power consumption).

Caveats

  • Power Consumption: Ultimate Performance can significantly increase power usage, especially on laptops.
  • Manual Selection: The script only makes Ultimate Performance available; the user must select it manually.
  • Limited Effectiveness: The impact of Game Mode and Ultimate Performance varies depending on hardware and games.

Important Note: Consider whether maximum performance is worth the increased power usage before running this script. If battery life is a priority, Ultimate Performance may not be the best choice.

KillAdobeCloud.cmd

This script is designed to forcefully close all running Adobe Creative Cloud applications and their associated background processes.

How It Works

  1. Checks for Administrator Privileges:
    • Ensures the script has the necessary permissions to terminate processes forcefully.
    • Prompts the user to elevate privileges if not running as administrator.
  2. Forcefully Terminates Adobe Processes:
    • Uses the taskkill /f /im command to end the following Adobe processes:
      • `Creative Cloud.exe`
      • `Adobe Desktop Service.exe`
      • `Adobe Crash Reporter.exe`
      • `CCXProcess.exe`
      • `AdobeIPCBroker.exe`
      • `AdobeNotificationClient.exe`
      • `Adobe Update Service.exe`
      • `Adobe Acrobat Update Service.exe`
      • `AGMService.exe` (Adobe Genuine Monitor Service)
      • `AGSService.exe` (Adobe Genuine Software Service)
      • `Adobe Genuine Software Integrity Service.exe`
      • `CoreSync.exe`
      • `CCLibrary.exe`
      • `Creative Cloud Helper.exe`
      • `Adobe Crash Processor.exe`
  3. Provides Feedback:
    • Prints a message indicating that all Adobe Creative Cloud applications have been closed.

Key Points

  • Forceful Termination: The /f (force) flag ensures processes are terminated, even if unresponsive.
  • Process List: Targets specific Adobe processes associated with Creative Cloud applications.
  • Potential Disruption: Forcefully ending processes may result in unsaved work loss. Use with caution.

Considerations

  • Why Force Close? Use when Adobe processes become unresponsive or problematic.
  • Alternatives: Try closing applications normally via Task Manager or Adobe's tools before using this script.
  • Regular Use: Not ideal for regular use, as it could disrupt your workflow. Address the root causes of Adobe issues directly.

PauseWUpdate.cmd

This script appears to be designed to pause Windows Update until the year 2050.

How It Works

  1. Checks for Administrator Privileges:
    • Ensures the script has the necessary permissions to modify system settings.
    • Prompts the user to elevate privileges if not running as administrator.
  2. Displays an Intro Animation:
    • Creates a stylized text animation that says "Powered by TEO SUSHI" followed by "STAY HOME #COVID19" using ANSI escape codes.
    • Credits the intro to "Ghost Spectre."
  3. Modifies Registry to Pause Windows Updates:
    • Uses the REG ADD command to add or modify several registry keys and values under HKLM\SOFTWARE\Microsoft\WindowsUpdate.
    • These changes set parameters indicating updates are paused until 2050.
  4. Provides Feedback:
    • Prints a message: "Your Windows Update now is pause until 2050!!!!!!"
    • Pauses for 4 seconds to allow the user to read the message.

Key Points

  • Administrator Rights: Required to modify the registry.
  • Pause Windows Updates: Attempts to disable automatic updates by manipulating registry settings. Effectiveness may vary depending on your Windows version and configuration.
  • Potential Risks: Modifying the registry incorrectly can lead to system instability or unexpected behavior.
  • Security Concerns: Disabling automatic updates can leave your system vulnerable to security threats. Keeping updates enabled is generally recommended for security and stability.

Important Considerations

  • Windows Update is Essential: Updates often contain crucial security patches and bug fixes. Disabling them for extended periods can expose your system to risks.
  • Alternative Options: If you're concerned about interruptions, configure active hours or set a metered connection to limit automatic downloads.
  • Registry Backup: Create a backup of the registry before making any changes to ensure you can restore it if needed.

Disclaimer: Disabling Windows Updates for an extended period is not recommended due to the potential security risks involved. Use this script at your own discretion and be aware of the consequences.

RestoreWifi.cmd

This script is designed to reset network settings on your Windows machine.

How It Works

  1. Checks for Administrator Privileges:
    • Ensures the script has the necessary permissions to modify network configurations.
    • Prompts the user to elevate privileges if not running as administrator.
  2. Resets Winsock and IP Settings:
    • Executes netsh winsock reset to reset the Winsock catalog for network communication.
    • Executes netsh int ip reset to reset TCP/IP settings to defaults.
  3. Releases and Renews IP Address:
    • Executes ipconfig /release to release your current IP address.
    • Executes ipconfig /renew to request a new IP address from the DHCP server.
  4. Flushes DNS Cache:
    • Executes ipconfig /flushdns to clear the DNS cache.
  5. Resets Wi-Fi Connection:
    • Disables and re-enables your Wi-Fi adapter using netsh interface set interface "Wi-Fi" admin=disable and netsh interface set interface "Wi-Fi" admin=enable.
  6. Provides Feedback:
    • Prints messages indicating the progress of each step.
    • Pauses at the end for user review.

Key Points

  • Administrator Rights: Required for executing network commands.
  • Resets Network Configuration: Helpful for network connectivity issues.
  • Temporary Disruption: Will temporarily disconnect you from the internet.
  • Wi-Fi Name: Replace "Wi-Fi" with your actual Wi-Fi interface name if it differs.
